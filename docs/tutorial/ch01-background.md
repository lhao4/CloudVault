# 第一章　项目背景与需求分析

> **本章目标**：搞清楚我们要做什么、为什么要做、和 v1 相比改进了什么。
>
> **本章不写代码**，是后续所有章节的基础认知。

---

## 1.1 项目来源

CloudVault v2 基于 v1 完整重构。

v1 是一个"能跑"的原型，用 Qt 的 `QTcpServer` 搭建服务端，功能基本跑通，但遗留了一批严重问题：

- 服务端依赖 Qt GUI 框架，部署到 Linux 服务器需要安装 Qt，极不合理
- 密码明文存储和传输，存在安全漏洞
- 数据库操作用字符串拼接 SQL，存在 SQL 注入风险
- 协议代码在客户端和服务端各写了一份，维护困难
- 线程模型使用错误（在非主线程操作 Qt 控件）

v2 的目标不是"代码能跑"，而是**工程可维护、可测试、可扩展**。

---

## 1.2 产品定位

面向小型团队（<50 人）的私有化部署系统，整合：

- **即时通信**：1 对 1 聊天、群聊、离线消息
- **文件管理**：上传、下载、目录操作、大文件分片
- **文件分享**：好友间直接分享文件

数据完全存储在自己的服务器，不依赖第三方云服务。

---

## 1.3 核心功能需求

### 用户系统
- 注册、登录、登出
- 个人信息（昵称、头像）

### 好友系统
- 搜索用户、查看在线用户
- 发送/接受/拒绝好友请求
- 好友列表、删除好友

### 聊天系统
- 1 对 1 实时聊天
- 离线消息（对方不在线时存库，上线后补投）
- 群聊（创建、加入、消息广播）

### 文件管理
- 目录浏览、新建、移动、删除、重命名、搜索
- 普通文件上传下载
- 大文件分片上传下载（支持断点续传，最大 20GB）
- 好友间文件分享

---

## 1.4 非功能需求

| 类型 | 要求 |
|------|------|
| **安全** | SHA-256 + salt 密码存储、预处理语句防 SQL 注入、TLS 加密传输、路径遍历防护 |
| **性能** | 支持 ≥50 并发连接，消息延迟 <100ms（局域网） |
| **可靠性** | 客户端异常断开后服务端正确清理会话 |
| **可维护性** | 模块化、可单元测试、有文档 |

---

## 1.5 v1 与 v2 的核心差异

| 方面 | v1（原版） | v2（本次重构） |
|------|-----------|--------------|
| 服务端框架 | Qt（QWidget + QTcpServer） | 纯 C++（epoll + std::thread） |
| 构建系统 | qmake + .pro 文件 | CMake 3.20+ |
| 协议代码 | 两端各写一份 | 单一 `common` 静态库 |
| 数据库操作 | 字符串拼接 SQL | 预处理语句（防注入） |
| 密码安全 | 明文存储和传输 | SHA-256 + salt，传输前客户端哈希 |
| 线程模型 | QRunnable（用法有误） | 主线程 epoll I/O + 工作线程业务 |
| 客户端架构 | 单例 + UI 直接操作网络 | 三层分离（network / service / ui） |
| 安全措施 | 无 TLS，无路径校验 | TLS + 路径遍历防护 + 输入校验 |
