# CloudHive 云巢 — PRD（产品需求文档）

> **版本**：v2.0 | **状态**：已确认

---

## 1. 产品概述

| 项目 | 内容 |
|------|------|
| 产品名称 | **CloudHive / 云巢** |
| 一句话描述 | 面向小型团队的私有化云文件管理与即时通讯系统 |
| 版本 | v2.0（基于 v1 完整重构） |
| 定位 | 面向生产环境的 C/S 架构桌面应用 |
| 目标用户规模 | <50 人 |
| 客户端平台 | Windows（Qt） |
| 服务端平台 | Linux（纯 C++，支持裸机与 Docker 部署） |

### 1.1 产品背景

v1 版本是基于 Qt 的原型验证，存在服务端依赖 GUI 框架、SQL 注入、明文密码、协议代码重复、线程模型错误等严重问题。v2 将：

- 服务端重写为纯 C++（脱离 Qt）
- 修复全部已知安全漏洞和 Bug
- 新增群聊、大文件分片、文件搜索等关键功能
- 全面工程化（CMake 构建、单元测试、文档、容器化部署）

### 1.2 核心价值

1. **私有化部署**：数据完全掌控在自己的服务器上，无需依赖第三方云服务
2. **文件+通讯一体化**：在一个客户端内完成文件管理、分享和即时沟通
3. **轻量化**：单机即可运行，适合小型工作室、实验室、小团队

## 4. 非功能需求

### 4.1 安全性

| ID | 要求 | 描述 |
|----|------|------|
| SEC-1 | 密码安全 | SHA-256 + 随机 salt 哈希存储，客户端发送前哈希，密码永不明文传输/存储 |
| SEC-2 | SQL 注入防护 | 全部数据库操作使用预处理语句（Prepared Statements） |
| SEC-3 | TLS 加密 | 客户端-服务端通信全程 TLS 加密（OpenSSL / QSslSocket） |
| SEC-4 | 路径遍历防护 | 文件操作严格校验路径，禁止 `..` 等越权访问 |
| SEC-5 | 输入校验 | 用户名/密码长度限制、文件名合法性检查、文件大小限制 |

### 4.2 性能

| 指标 | 目标 |
|------|------|
| 并发连接数 | ≥50 |
| 消息转发延迟 | <100ms（局域网内） |
| 文件上传吞吐 | 受限于网络带宽，非 CPU 瓶颈 |
| 服务端内存占用 | <200MB（空闲时） |

### 4.3 可靠性

| 要求 | 描述 |
|------|------|
| 连接断开处理 | 客户端异常断开后服务端正确清理会话和在线状态 |
| 上传中断恢复 | 大文件分片上传支持断点续传 |
| 数据库连接池 | 连接异常时自动重连 |

### 4.4 部署

| 方式 | 描述 |
|------|------|
| 裸机部署 | 单台 Linux 服务器，手动安装 MySQL，运行服务端二进制 |
| Docker 部署 | Docker Compose 一键启动（服务端 + MySQL），挂载数据卷 |
| 配置外部化 | 服务端通过 JSON 配置文件配置所有参数（端口、数据库、路径等） |

## 5. 大文件分片设计要点

> 这是 v2 的亮点功能，PRD 层面给出关键约束。

| 项目 | 规格 |
|------|------|
| 最大单文件 | 20GB |
| 分片大小 | 4MB（可配置） |
| 传输方式 | 客户端切片 → 逐片上传 → 服务端落盘 → 合并/直写 |
| 断点续传 | 客户端记录已上传分片序号，重连后从断点继续 |
| 校验 | 每片 MD5/CRC32 校验，整文件 SHA-256 校验 |
| 分布式扩展（预留） | 存储层抽象为接口，v2 实现本地文件系统后端，未来可扩展为 MinIO/S3 后端 |

## 6. 群聊设计要点

| 项目 | 规格 |
|------|------|
| 群上限 | 50 人/群（与系统规模一致） |
| 创建群 | 任何用户可创建，创建者为群主 |
| 加入方式 | 群主邀请 或 其他成员邀请 |
| 退出 | 成员可主动退出；群主退出则解散群 |
| 消息 | 文本消息广播给所有在线成员；离线成员通过离线消息表投递 |
| 管理 | v2 不做复杂管理（无禁言、踢人等），保持简单 |

## 7. 数据库概要

| 表 | 用途 |
|----|------|
| user_info | 用户账号、密码哈希、在线状态、个人信息、头像路径 |

## 8. 技术架构概要

| 层级 | 服务端 | 客户端 |
|------|--------|--------|
| 语言 | C++17 | C++17 + Qt 6 |
| 构建 | CMake 3.20+ | CMake 3.20+ |
| 网络 | POSIX socket + epoll | QTcpSocket |
| 协议 | 共享 common 库（二进制 PDU v2） | 共享 common 库 |
| 数据库 | MySQL + libmysqlclient | — |
| 存储 | std::filesystem（可扩展接口） | QFile |
| 日志 | spdlog | qDebug |
| 配置 | nlohmann/json | QJsonDocument |
| 密码 | OpenSSL SHA-256 | 共享 common 库 |
| 部署 | 裸机 / Docker | Windows 安装包 |

> 详细架构设计见 [architecture.md](./architecture.md)。

## 9. 版本范围

### v2.0 包含（本次交付）

- 全部 P0 功能
- 全部 P1 功能
- 安全加固（SEC-1 ~ SEC-5）
- 裸机 + Docker 部署
- 完整文档集（8 份技术文档）
- 单元测试覆盖率 ≥ 80%（common/ + server/handler/）

### 明确不含（v2 之后考虑）

- Web 客户端 / 移动端客户端
- 分布式存储后端（仅预留接口）
- 应用内管理后台
- 文件版本管理
- 端到端加密（E2EE）
- 视频/语音通话
- 文件预览（图片、文档等）
