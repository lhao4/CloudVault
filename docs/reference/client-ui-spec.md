# CloudVault 客户端 UI 统一规范

> 版本：v1.0
> 状态：可实施蓝图
> 平台：Windows（Qt 6 Widgets + QSS）

---

## 1. 目标与约束

本文档是 CloudVault Qt 客户端的唯一 UI 规范，替代旧的 `UI设计.md` 和 `ui-flow.md` 双文档并行维护方式。

文档目标不是输出理想化效果图，而是基于当前代码结构，为下一轮客户端 UI 重构提供可直接执行的设计与交互蓝图。

本轮设计约束如下：

- 技术栈保持 `Qt 6 Widgets + QSS`
- 不切换到 `QML`
- 不依赖协议新增即可完成基础 UI 重构
- 文档必须与当前类结构对齐，不虚报完成度
- 视觉方向采用“桌面 IM 强化版”：延续浅色基调，但提升层级、信息密度和专业感

---

## 2. 当前实现基线

### 2.1 当前核心类

客户端当前 UI 骨架已经具备以下结构：

| 区域 | 当前类名 | 当前职责 |
|------|----------|----------|
| 登录页 | `LoginWindow` | 登录 / 注册入口 |
| 主窗口壳层 | `MainWindow` | 导航、布局切换、服务协调 |
| 左侧上下文栏 | `SidebarPanel` | 当前主要承载联系人 / 会话列表 |
| 聊天主区 | `ChatPanel` | 单聊消息展示、群聊页承载 |
| 文件主区 | `FilePanel` | 文件列表、路径、操作区 |
| 右侧详情区 | `DetailPanel` | 联系人状态、文件摘要 |
| 个人资料页 | `ProfilePanel` | 资料编辑、退出登录 |
| 在线用户弹窗 | `OnlineUserDialog` | 当前“添加好友”入口 |
| 群组列表弹窗 | `GroupListDialog` | 群组选择与群会话入口 |
| 文件分享弹窗 | `ShareFileDialog` | 选择好友分享文件 |

### 2.2 当前布局事实

`MainWindow` 当前已经是四区壳层：

1. 最左侧图标导航栏 `iconBar`
2. 左侧上下文栏 `SidebarPanel`
3. 中央主工作区 `QStackedWidget`
4. 右侧详情区 `DetailPanel`

这意味着下一轮 UI 重构不需要推翻主窗口结构，只需要统一导航语义、内容切换规则和视觉层级。

### 2.3 当前完成度边界

- 单聊已经具备消息收发、历史加载、未读统计、空态切换
- 群组已经具备建群、群列表、群会话切换入口
- 群历史消息独立协议尚未真正打通，UI 需要预留但不能写成“完整已实现”
- 文件模块已具备上传、下载、刷新、搜索、重命名、移动、删除、分享入口
- 个人资料页已具备编辑与保存入口
- 断线横幅、基础状态提示已在主窗口存在

## 3. 目标信息架构

### 3.1 总体布局

客户端统一采用“1 条主导航 + 1 个上下文侧栏 + 1 个主工作区 + 1 个可收起详情区”的桌面 IM 布局。

```text
+----+----------------------+--------------------------------------+------------------+
|导航|     上下文侧栏       |              主工作区                |      详情区      |
+----+----------------------+--------------------------------------+------------------+
|头像| 会话 / 联系人 / 目录 | 聊天 / 联系人资料 / 文件内容 / 我的页 | 联系人/群/文件详情|
|消息| 搜索 + 筛选 + 动作   | 顶栏 + 主内容 + 底部操作区            | 可折叠/按场景隐藏  |
|联系人|                    |                                      |                  |
|文件|                      |                                      |                  |
|设置|                      |                                      |                  |
+----+----------------------+--------------------------------------+------------------+
```

### 3.2 主导航定义

主导航固定为以下 5 个入口：

| 入口 | 含义 | 切换结果 |
|------|------|----------|
| 头像 | 我的 | 打开 `ProfilePanel` |
| 消息 | 会话中心 | 上下文侧栏显示会话列表，主区显示 `ChatPanel` |
| 联系人 | 好友与群组 | 上下文侧栏显示好友 / 群组树，主区显示资料页 |
| 文件 | 文件管理 | 上下文侧栏显示目录/筛选/传输，主区显示 `FilePanel` |
| 设置 | 客户端设置 | 当前可先作为占位页或轻量弹窗 |

说明：

- “头像”不再只是按钮，而是“我的”一级入口
- “联系人”必须是独立主视图，不再以弹窗替代
- `OnlineUserDialog` 只保留为过渡实现，不应继续作为联系人体系核心

### 3.3 上下文侧栏定义

`SidebarPanel` 在目标态中被重新定义为“上下文侧栏容器”，而不是固定联系人列表。

不同主导航下的侧栏职责如下：

| 模式 | 侧栏内容 |
|------|----------|
| 消息 | 会话列表、搜索框、未读聚合、快捷动作 |
| 联系人 | 好友分组、群组分组、新朋友/建群入口 |
| 文件 | 目录树、筛选器、最近传输、共享入口 |
| 我的 | 默认隐藏 |
| 设置 | 默认隐藏或替换为设置导航 |

### 3.4 详情区定义

`DetailPanel` 统一定义为“详情区”，默认可见性由主视图决定：

| 模式 | 详情区行为 |
|------|------------|
| 消息 | 显示联系人资料 / 群资料 / 媒体摘要 |
| 联系人 | 显示好友资料、关系操作、共享文件摘要 |
| 文件 | 显示文件元数据、分享状态、传输状态 |
| 我的 | 默认隐藏 |
| 设置 | 默认隐藏 |

详情区必须支持按上下文收起，不能在所有模式下强制占据固定宽度。

## 4. 页面与组件规范

### 4.1 S-01 登录 / 注册页 `LoginWindow`

目标定位：

- 保持独立窗口
- 采用品牌头部 + 中央卡片 + 状态栏的轻量登录布局

核心组件：

- 登录 Tab：用户名、密码、登录按钮、服务端状态
- 注册 Tab：用户名、昵称、密码、确认密码、创建账号按钮
- 底部状态区域：连接状态、错误提示、可选服务器地址信息

交互规则：

- `Enter` 提交当前表单
- 登录失败时在表单内显示错误，不弹大量阻断弹窗
- 注册成功后自动回填登录用户名
- “昵称”必须在目标态中纳入统一资料模型，不再定义为纯展示字段

### 4.2 S-02 主窗口壳层 `MainWindow`

目标定位：

- 负责四区壳层布局
- 管理主导航切换
- 统一顶部连接横幅、全局提示、断线恢复状态

结构规则：

1. 左侧 `NavRail` 固定宽度，使用深色底
2. 中间区域由 `QSplitter` 控制侧栏、主区、详情区
3. 主工作区使用 `QStackedWidget` 切换消息 / 联系人详情 / 文件 / 我的 / 设置
4. 详情区支持场景化隐藏

状态规则：

- 正常态：显示当前主视图
- 断线态：顶部横幅提示，主功能保持可见但禁止误导性成功提示
- 加载态：在对应局部区域显示骨架或加载文案，不阻塞整个主窗口

### 4.3 S-03 消息视图

消息视图由“会话侧栏 + `ChatPanel` + 详情区”组成。

#### 侧栏内容

- 搜索框
- 会话列表
- 未读角标
- 最近消息预览
- 顶部快捷动作按钮

顶部动作建议：

- 新建聊天
- 添加好友
- 发起群聊

#### 会话列表项

每项展示：

- 头像或群组缩略标识
- 名称
- 最后一条消息预览
- 时间
- 未读数
- 在线状态或群人数摘要

#### `ChatPanel` 结构

- 顶栏：会话标题、在线状态或群简介、详情按钮
- 消息区：气泡流、日期分隔、文件卡片、发送状态
- 输入区：多行输入框、附件按钮、发送按钮

交互规则：

- `Enter` 发送
- `Shift+Enter` 换行
- 未选中会话时显示空态页
- 切换会话时清空当前未读并请求历史
- 群聊页必须使用与单聊一致的主结构，不能再写成“另一套独立占位大页”

#### 群聊状态说明

文档必须准确写为：

- 已支持群列表获取、建群、进入群会话
- 已支持群消息展示骨架
- 独立群历史协议仍需补齐
- 详情区中的成员列表、群设置、退群操作属于下一轮 UI 补强内容

### 4.4 S-04 联系人视图

联系人视图由“联系人侧栏 + 联系人主区 + 详情区”组成。

#### 侧栏内容

- 搜索框
- 新朋友入口
- 好友分组
- 群组分组
- 顶部动作按钮

顶部动作定义：

- 添加好友
- 发起群聊

#### 主区内容

选中好友时显示：

- 头像
- 用户名 / 昵称
- 在线状态
- 资料摘要
- 操作按钮：发送消息、分享文件、删除好友

选中群组时显示：

- 群名称
- 群人数摘要
- 群主标记
- 操作按钮：进入聊天、查看成员、退出群组

说明：

- 当前代码里联系人主区还没有独立 `ContactPanel`
- 第一轮可以先复用 `DetailPanel + ChatPanel` 组合承载联系人详情
- 文档中要把“联系人主视图”定义清楚，作为后续抽象单独组件的依据

### 4.5 S-05 文件视图 `FilePanel`

文件视图采用“目录 / 筛选侧栏 + 文件主区 + 文件详情区”的三段式布局。

主区包含：

- 顶栏：路径面包屑、上传、刷新、搜索
- 内容区：文件列表或网格
- 底部操作区：重命名、移动、删除、下载、分享
- 传输状态区：上传/下载中的任务摘要

交互规则：

- 双击目录进入下一级
- 支持返回上级
- 右键菜单与底部操作区保持一致
- 删除操作必须明确为危险操作
- 分享文件后应明确进入“生成文件消息卡片”的业务流

### 4.6 S-06 个人资料视图 `ProfilePanel`

目标定位：

- 作为头像入口对应的完整主视图
- 不再视为零散设置页

内容组成：

- 头像 / 用户名 / 昵称
- 基础资料编辑
- 当前账号标识
- 保存 / 取消
- 退出登录

资料更新必须与服务端资料模型保持一致，避免“UI 可编辑但服务端不持久化”的设计分叉。

### 4.7 弹窗体系

#### `OnlineUserDialog`

- 仅作为当前实现中的临时“添加好友”入口
- 目标态应弱化其在主流程中的核心地位

#### `GroupListDialog`

- 当前用于群组选择
- 目标态中逐步转为“群组快捷切换 / 管理弹窗”

#### `ShareFileDialog`

- 保持模态弹窗
- 支持选择好友或群组
- 成功后通过统一消息流反馈到聊天界面

## 5. 关键业务流程

### 5.1 启动与登录流程

```text
应用启动
  -> 创建 LoginWindow
  -> 用户登录成功
  -> App 创建 MainWindow
  -> 默认进入“消息”视图
  -> 拉取好友、群组、文件列表与个人资料
```

异常流程：

```text
网络断开
  -> MainWindow 显示顶部断线横幅
  -> 当前页面保留
  -> 后台自动重连
  -> 重连成功后刷新必要数据
```

### 5.2 消息与好友流程

添加好友：

```text
联系人视图
  -> 顶部动作“添加好友”
  -> 打开搜索/选择入口
  -> 发送好友请求
  -> 对方接受
  -> 双方进入好友列表
  -> 自动出现在消息会话候选中
```

发送单聊消息：

```text
消息视图
  -> 选中会话
  -> ChatPanel 输入内容
  -> Enter 发送
  -> 本地先行上屏
  -> 收到服务端回执/广播
  -> 刷新会话预览与未读状态
```

发起群聊：

```text
联系人视图
  -> 顶部动作“发起群聊”
  -> 选择好友并填写群名称
  -> 创建成功
  -> 自动进入群会话
  -> 后续可从消息视图继续访问
```

### 5.3 文件流程

上传文件：

```text
文件视图
  -> 点击上传
  -> 选择本地文件
  -> 进入传输状态区
  -> 上传完成
  -> 刷新当前目录
```

文件管理：

```text
选中文件
  -> 可执行重命名 / 移动 / 删除 / 下载 / 分享
  -> 危险操作需二次确认
  -> 操作完成后刷新列表与详情区
```

文件分享：

```text
文件视图选择文件
  -> 点击分享
  -> 打开 ShareFileDialog
  -> 选择好友或群组
  -> 生成文件消息卡片
  -> 对应会话中可见
```

## 6. 样式系统

### 6.1 视觉原则

- 保留浅色桌面风格
- 导航与内容层级强对比
- 列表密度高但不拥挤
- 重点状态可一眼区分：在线、未读、危险、断线、发送失败

### 6.2 基础 Token

当前 `style.qss` 的色值可继续沿用，并补足语义定义：

| Token | 色值 | 用途 |
|------|------|------|
| `--accent` | `#3B82F6` | 主按钮、激活态、自己的消息气泡 |
| `--accent-hover` | `#2563EB` | 主按钮悬停 |
| `--accent-soft` | `#EFF6FF` | 选中背景、轻提示 |
| `--bg` | `#F4F6F8` | 应用底色 |
| `--surface` | `#FFFFFF` | 面板背景 |
| `--surface2` | `#F0F2F5` | 输入框、次级容器 |
| `--border` | `#E2E6EA` | 边框、分割线 |
| `--text` | `#111827` | 主文字 |
| `--text2` | `#6B7280` | 次级文字 |
| `--text3` | `#9CA3AF` | 时间、占位符、弱提示 |
| `--online` | `#22C55E` | 在线 |
| `--danger` | `#EF4444` | 删除、错误、风险提示 |

### 6.3 尺寸建议

| 组件 | 目标尺寸 |
|------|----------|
| 主窗口最小尺寸 | `960 x 620` |
| 导航栏宽度 | `52px` |
| 上下文侧栏宽度 | `240px` |
| 详情区宽度 | `220px` |
| 列表项高度 | `60px - 68px` |
| 输入框高度 | `40px` |
| 图标按钮 | `30px - 36px` |

### 6.4 组件样式要求

#### 导航栏

- 深色背景，突出全局层级
- 激活项使用亮色前景 + 左侧指示条
- 头像入口与普通导航按钮视觉上要区分

#### 列表

- Hover 轻背景
- Selected 使用 `--accent-soft`
- 未读角标使用高饱和主色
- 群组和联系人项允许使用不同辅助元信息

#### 输入区

- 使用圆角浅底输入框
- Focus 时显示清晰边框变化
- 按钮组不堆叠过多无标签图标

#### 消息气泡

- 自己发送：主色底、白字
- 对方发送：浅灰底、深色字
- 文件卡片需与纯文本消息区分
- 群聊中左侧消息展示发送者名称

#### 详情区

- 作为次级信息面板，弱于主区但强于背景
- 标题、元信息、危险操作区域要有明显层级划分

## 7. 状态模型

所有核心页面至少统一支持以下状态：

| 状态 | 表现方式 |
|------|----------|
| 空状态 | 友好提示 + 引导操作 |
| 加载态 | 局部占位或加载文案 |
| 正常态 | 展示完整内容 |
| 错误态 | 区域内错误提示，可重试 |
| 断线态 | 顶部横幅 + 当前功能弱化提示 |

补充要求：

- 不使用大量阻断式弹窗代替页面内状态反馈
- 文件、消息、资料保存应优先使用局部状态提示
- 只有删除、退出、不可恢复操作使用强确认

## 8. 实施优先级

### 阶段 1：结构对齐

- 将主导航语义固定为“消息 / 联系人 / 文件 / 我 / 设置”
- 让联系人成为独立主视图，而不是主要依赖弹窗
- 让详情区支持按场景隐藏

### 阶段 2：消息体验补强

- 重构会话侧栏的信息层级
- 统一单聊和群聊的顶部栏与输入区
- 补全群聊页的可视状态，不再展示明显占位感

### 阶段 3：联系人与群组体验

- 将“添加好友 / 发起群聊”提升为联系人视图一级动作
- 增强好友详情与群资料页
- 明确新朋友、群组入口和关系操作区

### 阶段 4：文件体验补强

- 强化目录、传输、详情区三者关系
- 优化危险操作反馈
- 打通文件分享后的消息反馈体验

### 阶段 5：统一样式收口

- 整理 `style.qss` 的 token 和命名
- 统一列表、按钮、输入框、详情区和状态提示样式
- 清理旧布局遗留的视觉不一致问题
